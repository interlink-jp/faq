## 2. GitHub Pagesについてわかったこと

- **商用利用OK**（ECサイト・SaaSは不可）
- **独自ドメイン設定可能**（GitHub側は無料、DNS設定のみ）
- **無料枠：** ストレージ1GB、月間転送量100GB
- **静的サイト**なのでサーバー処理はできないが、JavaScript（キーワード検索・フィルター・アコーディオン等）は動く
- **AIチャット機能**はAPI中継サーバーが別途必要（GitHub Pagesだけでは不可）
- 企業の公式ドキュメントやFAQをGitHub Pagesで公開するのは一般的

## 3. AI組み込みの検討

### レベル1：意味検索（Embedding）
- ユーザーの質問と意味が近いFAQを検索して一覧表示
- コスト：**約0.01〜0.05円/1問**
- 既存の回答をそのまま返す（正確だが、複合的な質問への対応は弱い）

### レベル2：AI回答生成（RAG）
- 関連FAQを複数取得し、AIがまとめて回答を生成
- コスト：**約1〜10円/1問**（Haiku/Sonnetなら2〜5円が現実的）
- 複雑な質問に対応できるが、ハルシネーション（誤回答）のリスクあり
- 突き詰めるとチャットボットになり、FAQ一覧画面は不要に
- 実務ではハイブリッド（AI回答＋出典FAQ表示）が多い

## 4. FAQ整理へのAI活用

- 現状のFAQがスパゲッティ状態なら、**まずAIで整理するのが先**
- API不要（Claude Pro月額内で可能）
- URLを渡すだけでFAQデータを読み取れる（CSV書き出し不要）
- 出力：**3段階**で提供可能
  1. 現状分析レポート（重複・矛盾・古い情報の指摘）
  2. 整理案（統合・分割・削除・新規追加の提案と理由）
  3. 修正案（修正後のFAQ全体＋旧FAQ番号との対応表）
